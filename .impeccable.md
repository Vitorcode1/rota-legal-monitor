## Design Context

### Users

Três perfis chegam ao site em momentos distintos da jornada:

1. **Decidindo (topo de funil)**: Brasileiro que ouviu falar que dá para trabalhar no exterior mas ainda não sabe se é viável para o seu perfil. Precisa ser convencido com fatos antes de mergulhar nos detalhes.
2. **Comparando (meio de funil)**: Já quer ir, mas está escolhendo entre países. Quer ver critérios que importam para a sua situação lado a lado: dificuldade do visto, salário mínimo, autonomia permitida, idioma necessário.
3. **Em processo (fundo de funil)**: Escolheu o país, está juntando documentos. Usa o site como checklist e referência contínua, volta várias vezes.

O público é amplo: profissionais de TI, saúde, engenharia, trabalhadores autônomos, jovens em Working Holiday, pessoas em transição de carreira, qualquer brasileiro que queira trabalhar legalmente em outro país. Não há restrição de setor ou profissão.

Contexto de uso: pesquisa em casa, no celular ou desktop, provavelmente à noite. Ansiedade presente (decisão de vida grande). Baixa tolerância para juridiquês. Alta tolerância para informação densa se ela estiver organizada.

### Brand Personality

Três palavras: **confiável, direto, vivo**.

- Confiável: fontes linkadas, datas de atualização visíveis, limitações declaradas. Parece governo mas funciona de verdade.
- Direto: linguagem de guia de amigo, não de advogado de imigração. Explica o que importa para quem quer trabalhar, não para quem quer processar burocracia.
- Vivo: dados atualizados mensalmente de forma automática. O usuário sente que o site "respira".

O site deve evocar: competência tranquila. Não precisa impressionar, precisa gerar confiança.

### Aesthetic Direction

Dark mode. Canvas escuro (`#0a0a0a`), âmbar dourado (`#f0b429`) como único accent, sem segunda cor de marca. Tipografia Inter + JetBrains Mono. Inspiração visual: ClickHouse docs — denso, preciso, sem decoração.

Tema dark porque o perfil do usuário pesquisa de noite, ambiente de incerteza e pesquisa séria. Amarelo funciona como sinal de ação e destaque sem gritar.

Anti-referências: sites de agência de imigração (stock photos de família feliz, gradientes coloridos), portais gov.br (confuso, inchado), guias de blog informal (sem estrutura de dados).

### Design Principles

1. **Dados primeiro, narrativa depois.** O número ou fato concreto aparece antes da explicação. Uma stat grande antes de um parágrafo, não depois.
2. **Profundidade progressiva.** O resumo rápido aparece antes das abas técnicas. O usuário lê o que precisa e só aprofunda se quiser.
3. **Confiança visível.** Cada bloco de dado mostra quando foi atualizado e de onde veio. Sem dados flutuando no ar.
4. **Uma ação por tela.** Cada página tem um CTA principal. O resto é suporte.
5. **Amarelo é escasso.** Um uso de amarelo por seção, no máximo. A raridade é o que faz ele funcionar.

### Site Map

```
/ (Home)
├── /paises           Lista de países com filtros
│   └── /paises/[cc] Detalhe de país (5 abas)
├── /comparar         Comparador lado a lado (2-3 países)
├── /qual-pais        Questionário de recomendação
├── /historico        Timeline mensal de mudanças
├── /calculadora      Calculadora de reserva financeira
└── /sobre            Sobre o projeto e metodologia
```

### Sections per Page

#### Home
1. Hero — proposta de valor em 2 linhas + 2 CTAs (Comparar países / Qual país é o meu?)
2. Confiança — strip com: data de atualização, nº países, nº tipos de visto, "fontes oficiais"
3. Os três caminhos — 3 cards grandes mapeando os 3 perfis para as seções certas
4. Países em foco — top 5 destinos com card resumido (NL, PT, IE, DE, FR)
5. Como funciona — 3 passos: fontes → IA extrai → validado
6. Última atualização — excerpt do histórico mais recente
7. Footer

#### /paises (Lista)
1. Header com filtros — Schengen, dificuldade do visto, trabalho autônomo, Working Holiday disponível
2. Grid de países — cards com flag, nome, confiança, nº vistos, "atualizado X dias"
3. Legenda de confiança — o que significa Alta/Média/Baixa

#### /paises/[cc] (Detalhe)
1. Header sticky — flag + nome + badges (Schengen, Work Permit, Confiança)
2. Resumo rápido — dificuldade do visto, salário mínimo, autônomo permitido, prazo total estimado
3. Abas: Vistos | Para Brasileiros | Requisitos Gerais | Mudanças Recentes | Fontes

#### /comparar
1. Seletor — escolher 2 ou 3 países
2. Tabela comparativa — linhas: dificuldade, salário mínimo, prazo, Schengen, autônomo, família, idioma exigido
3. Destaque — qual país lidera em cada critério

#### /qual-pais
1. Intro — "Algumas perguntas, uma recomendação com justificativa"
2. Progressão — 1 pergunta por vez: área profissional, qualificação, inglês/idioma, urgência, idade, prioridade (salário vs facilidade vs idioma)
3. Resultado — país recomendado + justificativa + plano B + link para detalhe completo

#### /historico
1. Filtros — por país, por relevância (alta/média/baixa)
2. Timeline mensal — o que mudou em cada ciclo com indicador de impacto
3. Alerta — "receber aviso quando algo mudar" (email ou RSS)

#### /calculadora
1. Inputs — país destino, duração, estilo de vida (econômico/médio/confortável)
2. Output dinâmico — valor a juntar, quebrado por categoria (moradia, alimentação, transporte, documentação)
3. Premissas visíveis — quais dados foram usados e de quando são

#### /sobre
1. Por que existe — o problema que o site resolve
2. Como funciona — pipeline técnico em linguagem acessível
3. Frequência e confiança — quando atualiza, limitações declaradas
4. Contribuir — como reportar erro, como sugerir país
5. Contexto do projeto: ecossistema Vitorcode1
