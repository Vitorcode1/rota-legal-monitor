Como os dados chegam até você

01

Fontes visitadas

No dia 1 de cada mês, URLs oficiais de imigração de 10 países são visitadas automaticamente via GitHub Actions. Sem intervenção manual, sem conteúdo de segunda mão.

02

IA extrai e valida

Claude analisa o HTML e extrai vistos, requisitos, prazos e taxas em formato estruturado. Cada campo é validado contra um schema Zod antes de ser salvo. Falha na validação? Nada é publicado.

03

Publicado como JSON

Os dados ficam disponíveis via GitHub Pages com CORS aberto. O site consome direto, sem servidor intermediário. Cada JSON carrega a data exata da verificação.

.github/workflows/monthly-update.yml 
 1on:
 2  schedule:
 3    # Dia 1 de cada mês, 06:00 UTC
 4    - cron: '0 6 1 * *'
 5
 6jobs:
 7  update:
 8    runs-on: ubuntu-latest
 9    steps:
10      - name: Extrair todos os países
11        run: bun run extract
12
13      - name: Detectar mudanças
14        run: bun run diff
15
16      - name: Publicar no GitHub Pages
17        uses: peaceiris/actions-gh-pages@v4
18        with:
19          publish_dir: ./data/current
20          publish_branch: gh-pages

O código é open-source. Você pode inspecionar exatamente o que é extraído e como é validado antes de ser publicado.
